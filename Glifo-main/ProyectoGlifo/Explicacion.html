Estructura y funcionamiento de GLIFO

Una guía detallada sobre cómo está construida la plataforma, sus componentes y la lógica detrás de cada actividad interactiva.

📱

¿Cómo está construida la aplicación?

GLIFO es una plataforma educativa web desarrollada con tecnologías web estándar: HTML5, CSS3 y JavaScript puro (Vanilla JS). No utiliza frameworks externos, lo que garantiza un código ligero, rápido y fácil de modificar.

La aplicación se compone de tres páginas principales que trabajan de manera integrada para ofrecer una experiencia completa de aprendizaje lector.

Sistema de pestañas dinámicas en Actividades

La página de actividades utiliza JavaScript para cambiar el contenido sin recargar la página, con efecto de disolución.


// Función de transición entre actividades
function transitionToActivity(activityId) {
    const contentDiv = document.getElementById('activityContent');
    
    // Aplicar efecto fade-out
    contentDiv.classList.add('fade-out');
    
    setTimeout(() => {
        // Cargar el contenido según el botón seleccionado
        if (activityId === 'cuestionario') loadCuestionario();
        else if (activityId === 'dragdrop') loadDragDrop();
        else if (activityId === 'juego') loadJuego();
        else if (activityId === 'evaluacion') loadEvaluacion();
        else if (activityId === 'escritura') loadEscritura();
        else if (activityId === 'multimedia') loadMultimedia();
        
        // Quitar fade-out y aplicar fade-in
        contentDiv.classList.remove('fade-out');
        contentDiv.classList.add('fade-in');
        setTimeout(() => contentDiv.classList.remove('fade-in'), 500);
    }, 300);
    
    // Resaltar botón activo
    document.querySelectorAll('.activity-btn').forEach(btn => {
        btn.classList.remove('active');
        if (btn.getAttribute('data-activity') === activityId) {
            btn.classList.add('active');
        }
    });
}

🎯 ¿Cómo funciona?

  • data-activity: Atributo personalizado que identifica qué actividad cargar.
  • Efecto fade: Clases CSS que controlan la opacidad y transición.
  • setTimeout: Permite que la animación termine antes de cambiar el contenido.
  • Botón activo: Se resalta visualmente la actividad seleccionada.

Actividad de Arrastrar y Soltar (Drag & Drop)

Implementación nativa de drag & drop para asociar 30 libros con sus respectivos autores.


// Configuración del Drag & Drop
function initDragEvents() {
    const books = document.querySelectorAll('.drag-book-item');
    const zones = document.querySelectorAll('.drop-author-zone');
    
    books.forEach(book => {
        book.setAttribute('draggable', 'true');
        book.addEventListener('dragstart', (e) => {
            draggedBook = book;
            book.classList.add('dragging');
            e.dataTransfer.setData('text/plain', book.getAttribute('data-libro'));
        });
    });
    
    zones.forEach(zone => {
        zone.addEventListener('dragover', (e) => {
            e.preventDefault();
            if (zone.getAttribute('data-matched') !== 'true') {
                zone.classList.add('drag-over');
            }
        });
        
        zone.addEventListener('drop', (e) => {
            e.preventDefault();
            const libroNombre = draggedBook.getAttribute('data-libro');
            const autorCorrecto = draggedBook.getAttribute('data-autor');
            const autorZona = zone.getAttribute('data-autor');
            
            if (autorCorrecto === autorZona) {
                // Emparejamiento correcto
                zone.setAttribute('data-matched', 'true');
                draggedBook.remove();
                dragCompletados++;
                actualizarProgresoDrag();
                showToast("¡Correcto!");
            } else {
                showToast("Incorrecto, intenta de nuevo", true);
            }
        });
    });
}

🖱️ Explicación del Drag & Drop:

  • dragstart: Se ejecuta al comenzar a arrastrar, guarda el libro seleccionado.
  • dragover: Necesario para permitir el "drop", previene el comportamiento por defecto.
  • drop: Verifica si el libro corresponde al autor y actualiza la interfaz.
  • data-matched: Evita que un autor reciba múltiples libros.

Evaluación automática (Verdadero / Falso)

Corrección automática de 30 preguntas con generación de porcentaje y retroalimentación.


// Calcular evaluación
function calificarEvaluacion() {
    let aciertos = 0;
    
    for (let i = 0; i < evaluacionPreguntas.length; i++) {
        const selected = document.querySelector(`input[name="q${i}"]:checked`)?.value;
        const esCorrecto = (selected === 'true' && evaluacionPreguntas[i].esVerdadero) ||
                           (selected === 'false' && !evaluacionPreguntas[i].esVerdadero);
        if (esCorrecto) aciertos++;
    }
    
    const porcentaje = Math.round((aciertos / evaluacionPreguntas.length) * 100);
    let mensaje = "";
    
    if (porcentaje >= 80) mensaje = "¡Excelente! Dominas la literatura universal.";
    else if (porcentaje >= 60) mensaje = "¡Muy bien! Tienes buen conocimiento.";
    else mensaje = "Sigue practicando, revisa los conceptos.";
    
    mostrarResultados(aciertos, porcentaje, mensaje);
}

// Base de datos de preguntas
const evaluacionPreguntas = [
    { texto: "Don Quijote fue escrito por Cervantes.", esVerdadero: true },
    { texto: "Cien años de soledad fue escrita por Isabel Allende.", esVerdadero: false },
    // ... 30 preguntas en total
];

📊 Explicación de la evaluación:

  • Base de datos local: Array de objetos con pregunta y respuesta correcta.
  • Recorrido dinámico: Itera sobre todas las preguntas para contar aciertos.
  • Retroalimentación: Mensaje personalizado según porcentaje obtenido.

Escritura creativa con palabras clave

Generador aleatorio de palabras y evaluación automática del poema.


// Banco de palabras (60 palabras)
const WORD_BANK = [
    'amor', 'soledad', 'alegria', 'tristeza', 'sueño', 'nostalgia',
    'mar', 'viento', 'fuego', 'tierra', 'cielo', 'luna',
    'sombra', 'luz', 'espejo', 'camino', 'puerta', 'silencio',
    // ... 60 palabras en total
];

// Generar palabras aleatorias
function generateRandomKeywords(count = 4) {
    const shuffled = [...WORD_BANK];
    for (let i = shuffled.length - 1; i > 0; i--) {
        const j = Math.floor(Math.random() * (i + 1));
        [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
    }
    return shuffled.slice(0, count);
}

// Evaluar poema
function evaluarPoema(poema, palabrasClave) {
    let puntuacion = 0;
    let palabrasUsadas = 0;
    
    palabrasClave.forEach(palabra => {
        if (poema.toLowerCase().includes(palabra.toLowerCase())) {
            palabrasUsadas++;
        }
    });
    
    puntuacion += palabrasUsadas; // 1 punto por cada palabra clave usada
    
    const lineas = poema.split('\n').filter(l => l.trim().length > 0).length;
    if (lineas >= 3) puntuacion += 2; // +2 por buena estructura
    
    return { puntuacion, palabrasUsadas };
}

✍️ Explicación de la escritura creativa:

  • Banco de 60 palabras: Categorizadas en emociones, naturaleza, conceptos abstractos, etc.
  • Generación aleatoria: Selecciona 4 palabras diferentes cada vez.
  • Evaluación automática: Califica según uso de palabras, longitud y estructura.

Nubes informativas interactivas

Sistema de ventanas emergentes que muestran información detallada al hacer clic en los beneficios.


// Sistema de nubes informativas
function showCloud(benefitId) {
    const content = beneficiosContent[benefitId];
    if (content) {
        cloudTitle.textContent = content.titulo;
        cloudText.textContent = content.texto;
        overlay.classList.add('show');
        cloud.classList.add('show');
    }
}

// Eventos de los beneficios
document.querySelectorAll('.benefit-item').forEach(item => {
    item.addEventListener('click', () => {
        const benefitId = item.getAttribute('data-benefit');
        showCloud(benefitId);
    });
});

// Cerrar con ESC o click fuera
overlay.addEventListener('click', closeCloud);
document.addEventListener('keydown', (e) => {
    if (e.key === 'Escape' && overlay.classList.contains('show')) {
        closeCloud();
    }
});

💡 Explicación de las nubes:

  • Overlay: Fondo semitransparente que enfoca la atención.
  • data-benefit: Identifica qué contenido mostrar.
  • Tecla ESC: Accesibilidad para cerrar la ventana.
